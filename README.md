# Спектральный анализ представлений в CNN и трансформерах

Репозиторий содержит код для магистерской диссертации на тему **«Спектральные характеристики представлений в CNN и трансформерах»**.  
В работе исследуется, как распределение спектральной энергии по пространственным частотам меняется с глубиной в трёх семействах архитектур: свёрточной **ResNet**, визуальном трансформере **ViT** и иерархическом трансформере **Swin**.

## 🧠 Краткое содержание

Центральный вклад работы — разработка единого протокола спектрального анализа, позволяющего корректно сопоставлять разнородные архитектуры.  
Работа включает контрольные эксперименты, отделяющие эволюцию признаков от тривиального эффекта уменьшения разрешения.

Эксперименты на подвыборке ImageNet показали:
- **ResNet** — наиболее устойчивое низкочастотное сжатие;
- **ViT** на фиксированной решётке патчей — немонотонная динамика;
- **Swin** — приходит к низкочастотному финальному профилю, но во многом за счёт иерархического понижения разрешения.

Подробные результаты и выводы — в [thesis.pdf](thesis.pdf).

## 📊 Данные

Эксперименты проводились на подвыборке **ImageNet** из **10 000 изображений** .  
Для воспроизводимости результатов настоятельно рекомендуется использовать именно эту стандартную выборку.

**Перед запуском** укажите путь к данным в переменной `IMAGENET_DIR` в начале каждого ноутбука.  
Если полный ImageNet недоступен, можно использовать любой другой набор данных, но в таком случае результаты будут отличаться от представленных в диссертации.

## 🚀 Запуск

Ноутбуки можно выполнять в любом порядке, однако логическая последовательность такова:

1. **01–03** – генерация спектров для каждой архитектуры (занимает от 1 до 6 часов в зависимости от GPU и размера модели).
2. **04** – построение сводных графиков и таблиц на основе сохранённых спектров.

### Настройка путей

В начале каждого ноутбука определены переменные:
- `IMAGENET_DIR` – путь к данным ImageNet;
- `SAVE_PATH` – корневая папка для результатов (по умолчанию `results/`);
- `FIG_PATH`, `TABLE_PATH`, `SPECTRA_PATH` – подпапки для графиков, таблиц и сырых спектров.

Вы можете изменить эти пути по своему усмотрению.

### Запуск в среде без GPU

Если CUDA недоступна, ноутбуки автоматически переключатся на CPU — это увеличит время выполнения, но позволит запустить эксперименты.

### Результаты

После выполнения всех ноутбуков в папке `results/` появятся:
- `spectra/` – `.npz` файлы с сырыми спектрами каждого слоя (для повторного использования);
- `tables/` – `.csv` и `.tex` таблицы с метриками;
- `figures/` – `.png` графики спектральных профилей.

Все пути можно изменить в начале соответствующих ноутбуков.
